Driver for the control of a local installation of the OMXPlayer.

Make sure OMXPlayer is installed: 

sudo apt-get install omxplayer
OMXPlayer on Raspberry Pi requires the ARM side libraries for interfacing to the GPU. Those libraries are located in /opt/vc/lib.
The following libraries are required:
  • libWFC.so
  • libEGL.so
  • libbcm_host.so
  • libvchiq_arm.so
  • libvcos.so
  • libGLESv2.so
  • libopenmaxil.so
  • libbrcmGLESv2.so
  • libbrcmEGL.so
In case they are missing: 
sudo mkdir -p /opt/vc/lib
cd /opt/vc/lib
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libWFC.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libEGL.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libbcm_host.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libvchiq_arm.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libvcos.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libGLESv2.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libopenmaxil.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libbrcmGLESv2.so
sudo wget https://raw.githubusercontent.com/raspberrypi/firmware/master/hardfp/opt/vc/lib/libbrcmEGL.so
sudo ldconfig
Test your OMXPlayer installations running: 
sudo omxplayer <file>
To enable HDMI output at startup even when not connected, edit the file /boot/config.txt uncommenting the lines: 
hdmi_force_hotplug=1
hdmi_drive=2
